AppSheet 入門 ／ 寫給事務所行政的你

用 AppSheet 做出
自己的小工具，
不用會寫程式

AppSheet 是 Google 出的工具，可以把一份試算表，變成手機上能用的小軟體（像表單、清單、簽核那種）。這份說明把它拆成五個由下往上的階段——就像蓋房子，先打地基、再砌牆、最後裝潢上鎖。

先說在前面：你不需要會寫程式。看不懂的英文字都會放在小字、講解時一律用白話。看完你會知道「這東西在做什麼、卡住時問題出在哪」，剩下複雜的部分，可以交給 AI 幫忙。
怎麼讀：從最下面那層往上看 怎麼用：點一層 → 再點裡面的小標籤看例子
操作說明 ① 點任何一層，會往下展開。② 在「裡面有哪些東西」那區，點任一個有 + 的標籤，會跳出一張小卡片，告訴你：這是什麼、舉個例子、實際在哪裡點/要填什麼。灰底程式碼的右上角有「複製」可以直接拿去用。

第1

第一層：你的資料（地基）

DATA
一切的根基。AppSheet 自己不存資料，它只是幫你的試算表「裝上一個好用的介面」。
›
這層在做什麼

你的資料其實一直放在 Google 試算表裡，AppSheet 只是讀它、然後變成漂亮的畫面。所以你在試算表改了欄位，記得回 AppSheet 按一下「重新讀取」，它才會知道。

用生活比喻：試算表＝一個檔案櫃，每個抽屜（分頁）放一種資料。AppSheet＝你貼在櫃子外面、方便大家拿取的那張「使用說明＋取件窗口」。
裡面有哪些東西（點開看例子）
一張資料表 Table 欄位的種類 Type 每筆的編號 Key 表跟表的連結 Ref
一張資料表Table
白話講
就是試算表的一個分頁。一橫排是一筆資料，一直行是一個欄位。
舉個例子
專案表：每一排是一個案子（案名、業主、進度、金額）。請款表：每一排是一筆請款。
小建議
一個主題開一張表就好，別把所有東西塞進同一張——以後會很難整理。
欄位的種類Column Type
白話講
告訴系統「這格要填的是哪一種東西」，它就會給對應的輸入方式。
常用的幾種
文字、數字、日期、下拉選單（從幾個固定選項挑）、照片、地點。
舉個例子
「案件進度」設成下拉選單（規劃／設計／施工／結案）→ 大家就只能用選的，不會有人打錯字、各寫各的。
在哪裡設
左邊選 Data 資料 → 點那張表 → 找到那一欄 → 改它的「種類」
每筆的編號Key 主鍵
白話講
每一筆資料要有一個「不會重複的身分證號」，系統靠它認得是哪一筆。
為什麼重要
別拿「案名」當編號（會改名、會重複，一改就亂掉）。最好讓系統自動發一組看不見的流水號。
怎麼讓它自動發號（複製貼上就好）
UNIQUEID()複製
把這串貼到那欄的「預設值」欄位，系統就會自動產生編號。不用懂它怎麼運作，照貼即可。
表跟表的連結Ref
白話講
把兩張表「掛勾」在一起。掛好後，系統會自動幫你做選單跟對應清單。
舉個例子
請款表裡有一欄「屬於哪個案子」，掛到專案表。之後打開某個案子，下面就自動列出它所有的請款，不用自己找。
在哪裡設
把「屬於哪個案子」這欄的種類，改成 Ref 連結，再選要連到「專案表」
新手最容易卡的地方
忘了設「編號」，或拿會變的東西（人名、案名）當編號。記得：用系統自動發的流水號當身分證最安全。
這層之後可以怎麼「請 AI 幫忙」

這層的資料，是 AI 最容易幫上忙的地方。串好之後，你可以用講的請 AI 幫你查、幫你填、幫你整理資料，不用一筆一筆手動敲。（背後的技術名稱叫 API，你不用記。）

第2

第二層：自動算、自動篩（水電管線）

LOGIC
讓資料會自己動。例如自動算金額、自動帶今天日期、自動只給某人看到他的案子。
›
這層在做什麼

這層就是「自動幫你做計算跟篩選」。寫法很像 Excel 的公式——如果你會在 Excel 打 =A1*1.05，這層就難不倒你。複雜的公式都可以請 AI 直接幫你寫好，你貼上就行。

用生活比喻：就像在表格裡設好公式，金額會自己加總；這層讓你的小工具也有同樣的「自動算」能力。
裡面有哪些東西（點開看例子）
自動算的欄位 App Formula 預設帶入的值 Initial Value 篩選出一群 Slice 擋住亂填 Valid_If
自動算的欄位App Formula
白話講
設定後，這格「永遠由公式自動算」，人改不動，確保不會算錯。
舉個例子
「含稅金額」＝未稅 × 1.05，永遠自動算，沒人能手動改。
公式長這樣（複製貼上）
[未稅金額] * 1.05複製
預設帶入的值Initial Value
白話講
只是「先幫你填好預設」，你之後還是可以改。跟上面那個「改不動」不一樣。
舉個例子
新增一筆時，「日期」自動帶今天、「填表人」自動帶現在登入的這個人。
公式長這樣（複製貼上）
# 日期欄，自動帶今天 TODAY() # 填表人欄，自動帶登入者 USEREMAIL()複製
篩選出一群Slice 切片
白話講
把一張大表「篩出一小群」，存成一個固定的篩選。像 Excel 的篩選，但可以存起來重複用。
舉個例子
做一個「我負責的案子」，畫面就只顯示登入者自己的案，不會看到一大堆別人的。
篩選條件這樣寫（複製貼上）
[負責人] = USEREMAIL()複製
意思是：只留下「負責人＝現在登入這個人」的資料。
擋住亂填Valid_If
白話講
設一個規則，填得不對就送不出去，並跳提示。減少資料出錯。
舉個例子
「金額」一定要大於 0，否則表單不讓送出。
規則這樣寫（複製貼上）
[金額] > 0複製
新手最容易卡的地方
搞混「自動算（改不動）」跟「預設值（可以改）」。簡單記：要鎖死的用前者，只是想幫忙先填的用後者。
這層之後可以怎麼「請 AI 幫忙」

這層的公式，AI 最會幫你寫。你只要用講的：「我想要含稅自動算」「我只想看到自己的案子」，請 AI 給你公式，你再貼進去就好。不用自己背任何函數。

第3

第三層：畫面長什麼樣（裝潢）

VIEWS
使用者打開 App 看到的畫面。同一份資料，可以做成清單、表單、地圖等不同樣子。
›
這層在做什麼

決定大家「看到的畫面」。同一份資料可以擺成不同樣子，挑最順手的給同事用。

用生活比喻：同一批照片，你可以做成相簿、做成清單、貼在地圖上——資料沒變，只是「呈現方式」換了。
裡面有哪些東西（點開看例子）
清單畫面 Table / Deck 填寫表單 Form 單筆詳情 Detail 總覽看板 Dashboard
清單畫面Table / Deck
白話講
把很多筆資料排成一張清單。可以是純文字表格，也可以是有縮圖的卡片。
舉個例子
「進行中的案子」做成卡片清單，每張卡顯示案名＋業主＋照片。
在哪裡做
左邊 Views 畫面 → 新增一個 → 選資料來源 → 挑樣式
填寫表單Form
白話講
給大家填資料的畫面。它不是另一份資料，只是「寫進某張表」的入口。
舉個例子
同事用手機填「報修單」：選地點、拍照、寫狀況、送出。
小技巧
可以設「某欄選了某選項，才跳出另一欄」——例如選了「不合格」才要填「原因」
單筆詳情Detail
白話講
點清單裡某一筆後，看到那一筆的完整內容。
舉個例子
點某個案子，看到它全部欄位，下面還自動列出它的所有請款紀錄。
總覽看板Dashboard
白話講
把好幾個畫面併在同一頁，一眼看全局。適合用電腦看。
舉個例子
左邊案子清單、右邊該案詳情，點左邊右邊就跟著換，掌控全部進度。
新手最容易卡的地方
想只顯示「篩過的資料」，卻直接放整張表。記得先在第二層做好「篩選（Slice）」，再把畫面接上去。
這層之後可以怎麼「請 AI 幫忙」

畫面的排版要在編輯器自己點，但你可以請 AI 當設計顧問：「我有這些資料，該做哪幾個畫面比較好用？」它會給你一份照著做的清單。

第4

第四層：按鈕跟自動通知（電器開關）

ACTIONS
讓 App 會「做事」。例如一顆按鈕標記完成；或資料一變，就自動寄信通知。
›
這層在做什麼

分兩種：一種是「要人按」的按鈕；一種是「自動發生」的通知（沒人按，條件到了它自己跑）。

用生活比喻：按鈕＝電燈開關（你按它才亮）；自動通知＝感應燈（有人經過就自動亮）。
裡面有哪些東西（點開看例子）
要人按的按鈕 Action 自動通知 Bot / Automation 接到別的系統 Webhook
要人按的按鈕Action
白話講
使用者按一下就做一件事：改狀態、跳到別頁、打電話、開連結。
舉個例子
案子上一顆「結案」按鈕，按下去把狀態改成「結案」、並記下時間。
背後設的值（複製貼上）
# 狀態欄改成 "結案" # 結案時間欄填現在 NOW()複製
自動通知Bot / Automation
白話講
沒人按、條件到了自己跑。一個自動通知＝「什麼時候發生」＋「要做什麼」。
舉個例子
當一筆請款被標成「待簽核」→ 自動寄 Email 通知主管。或每週一早上 8 點，自動寄出本週待辦。
在哪裡做
左邊 Automation 自動化 → 新增 → 設「何時觸發」→ 設「要做什麼（寄信）」
接到別的系統Webhook
白話講
自動通知的進階版：資料一變，就把它「丟給」另一套系統處理。
舉個例子
填好一筆紀錄，就自動同步存進事務所的 NAS、或寫進 Notion。
這代表什麼
這就是把 AppSheet 接進事務所其他工具的關鍵接點。設定通常會請懂的人或 AI 幫忙，你知道「有這個能力」就夠了。
新手最容易卡的地方
分不清「按鈕（要人按）」跟「自動通知（自己跑）」。還有：自動通知的條件沒設好，會狂寄信轟炸大家。
這層之後可以怎麼「請 AI 幫忙」

這層可以雙向：AI 能幫你觸發 App 裡的動作；App 也能在事情發生時自動通知 AI 或其他系統接手。這是把整個事務所流程串起來、減少手動轉貼的關鍵。

第5

第五層：權限跟上線（門鎖與鑰匙）

SECURITY
誰能看到什麼、誰能改什麼，以及怎麼正式發給大家用。
›
這層在做什麼

做好不等於能用。這層決定權限（誰看得到哪些資料）跟「正式上線」。沒設好就分享出去，等於資料對所有人全開。

用生活比喻：就像辦公室的門禁卡——每個人刷卡只能進自己該進的區域，不是一張卡開全棟。
裡面有哪些東西（點開看例子）
誰能看哪些 Security Filter 正式上線 Deploy 給 AI 的鑰匙 API Key
誰能看哪些Security Filter
白話講
用一條規則，決定「每個人只能看到他該看的那幾筆」。
舉個例子
設計師只看到自己負責的案；行政可以看全部。
規則這樣寫（複製貼上）
# 只給看「負責人是自己」的資料 [負責人] = USEREMAIL()複製
正式上線Deploy
白話講
把 App 從「還在試做」正式發布成「大家能用」。系統會先幫你檢查有沒有問題。
舉個例子
內部測過沒問題，正式發給全事務所同事用。
在哪裡做
左邊 Manage 管理 → Deploy 部署 → 照它列的清單修一修 → 正式上線
給 AI 的鑰匙API Key
白話講
想讓 AI 或別的程式連進這個 App，要先在這裡「打開門、拿一把鑰匙」給它。
在哪裡拿
左邊 Settings 設定 → Integrations 整合 → 開啟 API → 複製那兩串代碼
重要提醒
這把鑰匙＝資料的存取權，跟銀行密碼一樣。不要貼到群組、不要外流。交給協助你的人時，請對方妥善保管。
新手最容易卡的地方
以為「沒給連結別人就看不到」。其實要靠上面的「誰能看哪些」規則才擋得住。還在「試做」狀態就分享，會有人數限制。
這層之後可以怎麼「請 AI 幫忙」

這層提供 AI 連進來的那把鑰匙。拿到鑰匙後，前面四層講的「請 AI 幫查、幫填、幫通知」才能真的運作。

一句話總結 這五層就像蓋房子：先有資料（地基）、再讓它會自動算（管線）、做出好看的畫面（裝潢）、加上按鈕跟通知（開關）、最後鎖好門再交鑰匙（門禁）。每一層都可以請 AI 幫忙，你負責想清楚要什麼就好。

那「請 AI 幫忙」到底是什麼意思？

你可能聽過 MCP 這個詞。它聽起來很可怕，但其實意思很單純：

MCP 就是一座「橋」——讓 AI（像 Claude）能跟你的 AppSheet 對話。有了這座橋，你就能用講的請 AI 幫你查資料、填資料、跑通知，不用自己一筆筆操作。

你不需要自己蓋這座橋。蓋橋是技術人員（或 AI 自己）的工作。你的角色是：把這五層的概念搞懂，知道自己想要什麼，剩下的交給 AI。

幫你先把「能」跟「不能」講清楚，免得期待落空

  • AI 能幫你做的：查資料、填資料、批次新增、按下按鈕觸發動作、自動發通知。這些都能自動化。
  • AI 目前還不能直接做的：幫你「畫畫面、寫公式、改整個結構」——這些還是要在編輯器手動點。但 AI 可以告訴你怎麼點、把公式寫好給你貼。
  • 所以最務實的分工：你（或請技術同事）把骨架在編輯器搭好；之後的日常跑資料、跑通知交給 AI。
1

先把這五層看懂

知道每層在做什麼、卡住時問題在哪。看完這頁你已經做到了。

2

搭一個最小的骨架

用一張表（例如報修單），做出表單＋清單。卡住就請 AI 給步驟。

3

用講的描述需求，請 AI 給公式，你貼進去。不用背任何函數。

4

需要自動化時，找人接「橋」

要 AI 自動跑資料時，請技術同事幫你拿鑰匙、接好那座橋。

5

上線前鎖好權限

回第五層設好「誰能看哪些」，鑰匙妥善保管，再正式發給大家。
